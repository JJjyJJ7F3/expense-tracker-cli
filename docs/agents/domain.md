# Domain Docs

工程技能在探索代码库前，应按本文件约定读取领域文档和架构决策。

## 探索前读取

- 根目录 `CONTEXT.md`
- `docs/adr/` 中与当前工作相关的 ADR

如果这些文件不存在，继续执行即可，不需要主动提示缺失，也不要提前建议创建。相关技能会在需要沉淀术语或决策时再创建它们。

## 文件结构

本仓库使用 single-context 布局：

```text
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-example.md
│   └── 0002-example.md
└── src/
```

## 使用统一领域词汇

当输出内容需要命名领域概念时，例如 issue 标题、重构建议、问题假设、测试名称，应优先使用 `CONTEXT.md` 中定义的术语。

如果需要的概念还没有出现在 glossary 中，说明可能存在术语空缺。此时应记录这个空缺，交给后续领域建模工作处理。

## 标明 ADR 冲突

如果输出内容与已有 ADR 冲突，必须明确指出冲突，而不是静默覆盖既有决策。

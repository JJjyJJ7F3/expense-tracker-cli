# ADR-0010: v1 删除交易使用物理删除

## 状态

已接受

## 背景

Expense Tracker v1 需要允许用户修正录入错误。由于 v1 不支持编辑交易，删除交易是修正错误数据的主要方式。

可以选择软删除，通过 `deletedAt` 或类似字段保留历史记录；也可以选择物理删除，直接从 CSV 文件中移除记录。v1 没有审计、恢复、同步或多用户需求，软删除会让列表和汇总都必须额外过滤已删除记录，增加不必要复杂度。

## 决策

v1 删除交易使用物理删除。

删除规则：

- `expense delete <id>` 从 CSV 存储文件中移除对应记录。
- 删除成功后重写 CSV 文件。
- 删除不存在的 `id` 时命令失败，并提示未找到对应交易。
- 删除成功后输出被删除交易的简短信息，方便用户确认。
- v1 不增加 `deletedAt` 或软删除标记。

## 后果

好处：

- 删除语义简单，符合 v1 复杂度。
- 列表和汇总不需要过滤软删除记录。
- CSV 文件保持只包含当前有效交易。

代价：

- 删除后无法从应用数据中恢复该交易。
- 没有删除审计记录。
- 如果后续需要恢复、审计或同步能力，需要重新设计删除模型。

# ADR-0001: v1 支持删除交易但不支持编辑交易

## 状态

已接受

## 背景

Expense Tracker v1 需要形成一个可用的命令行闭环：新增交易、本地保存、查看交易、删除错误记录，并查看周汇总和月汇总。

命令行录入容易出现错误。如果完全不能修正记录，用户无法信任后续汇总结果。但完整编辑功能会引入字段级更新、交互式修改、局部校验和更复杂的命令设计。

## 决策

v1 支持删除交易，但不支持编辑交易。

当用户录入错误时，使用删除命令移除错误交易，再通过新增命令录入正确交易。

## 后果

好处：

- v1 仍然可以修正错误数据，保证汇总可信。
- 命令设计更简单，适合作为练习项目的第一版。
- 领域逻辑可以先聚焦在交易建模、校验、持久化和汇总计算上。

代价：

- 用户修正单个字段时需要删除并重新新增。
- 后续如果加入编辑功能，需要补充字段更新规则和测试。

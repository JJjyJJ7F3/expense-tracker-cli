# ADR-0006: 金额使用 BigDecimal 并固定两位小数规则

## 状态

已接受

## 背景

Expense Tracker 需要对收入、支出和净收入做汇总计算。金额如果使用 `double` 或 `float`，可能出现浮点精度误差，导致汇总结果不可信。

v1 不处理币种，目标是先把金额输入、校验、存储和展示规则固定下来。

## 决策

v1 金额使用 Java `BigDecimal` 表示。

金额规则：

- 输入金额必须大于 `0`。
- 输入金额最多保留 2 位小数。
- 输出金额统一显示 2 位小数。
- v1 不处理币种，金额是无币种数值。
- 实现中不得使用 `double` 或 `float` 参与金额计算。

## 后果

好处：

- 汇总计算不会受到浮点误差影响。
- 输入和输出格式稳定，便于测试。
- v1 避免引入币种、汇率和本地化格式问题。

代价：

- 解析和格式化金额需要显式处理 scale 和舍入规则。
- 如果后续支持多币种，需要新增币种字段和更完整的金额模型。

# ADR-0003: v1 使用固定交易字段

## 状态

已接受

## 背景

Expense Tracker v1 需要支持新增、列表、删除、周汇总和月汇总。交易数据会存储在 CSV 文件中，因此字段集合需要稳定，避免实现过程中不断扩大范围。

v1 的目标是练习 Java 基础、文件 I/O、校验、汇总和领域逻辑分离，而不是覆盖完整个人财务管理软件的所有概念。

## 决策

v1 每条交易固定使用以下字段：

```text
id,type,amount,category,date,note,createdAt
```

字段含义：

- `id`：唯一标识，用于删除交易。
- `type`：交易类型，只能是 `income` 或 `expense`。
- `amount`：正数金额。
- `category`：分类，不能为空。
- `date`：交易发生日期，用于周汇总和月汇总。
- `note`：可选备注。
- `createdAt`：记录创建时间，用于审计和默认排序。

v1 不加入账户、币种、标签、多用户等字段。

## 后果

好处：

- CSV 表头稳定，导入导出和测试更直接。
- 删除、列表和汇总都能基于同一组字段实现。
- 避免 v1 范围扩大到完整财务系统。

代价：

- v1 不能区分多个账户或币种。
- v1 不能通过标签表达多维分类。
- 后续如果加入账户、币种或标签，需要设计数据迁移或兼容旧 CSV 表头。

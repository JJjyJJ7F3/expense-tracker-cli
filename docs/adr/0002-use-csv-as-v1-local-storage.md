# ADR-0002: v1 使用 CSV 作为本地存储格式

## 状态

已接受

## 背景

Expense Tracker v1 需要把交易数据保存在本地文件中，并支持 CSV 导入和导出。

项目目标是练习 Java 基础、文件 I/O、解析校验、汇总计算和领域逻辑分离。CSV 文件结构简单，便于人工检查，也能直接服务导入导出功能。

## 决策

v1 使用 CSV 作为主存储格式。

应用的新增、删除、列表、周汇总和月汇总都以 CSV 存储文件作为数据源。

## 后果

好处：

- 存储格式和 CSV 导入导出功能保持一致。
- 便于练习 Java 文件 I/O、解析和校验。
- 数据文件可以被用户直接打开检查。
- v1 不需要引入数据库或 JSON 依赖。

代价：

- CSV 解析必须正确处理逗号、引号和换行转义。
- 实现时不能用简单的 `split(",")` 解析记录。
- 如果后续需要更复杂的数据结构，可能需要迁移到 JSON、SQLite 或其他存储格式。

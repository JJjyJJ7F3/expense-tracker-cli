# PRD: Expense Tracker v1

Status: ready-for-agent

## Problem Statement

用户需要一个轻量的命令行工具，在本地记录日常收入和支出，并能快速查看周汇总和月汇总。

当前项目是一个 Java 练习项目，目标不是构建完整个人财务管理系统，而是通过一个真实但可控的 CLI Expense Tracker 练习 Java 基础、文件 I/O、测试和设计分层。v1 需要形成一个可用闭环：用户能新增交易、持久化到本地、查看列表、删除错误交易，并根据交易数据生成可信的周汇总和月汇总。

如果没有明确的数据模型、金额规则、日期范围规则和存储边界，后续实现容易变成一组松散命令，汇总结果也难以测试和信任。

## Solution

构建一个 Java 实现的非交互式 CLI Expense Tracker。

v1 提供以下能力：

- 通过命令新增收入或支出交易。
- 将交易持久化到本地 CSV 存储文件。
- 查看最近交易列表，用于核对录入结果。
- 通过 `id` 物理删除录错的交易。
- 查看 ISO 周汇总和自然月汇总。
- 对交易类型、金额、分类、日期等输入执行基础校验。
- 提供帮助信息和清晰错误提示。

v1 使用固定交易字段：`id`、`type`、`amount`、`category`、`date`、`note`、`createdAt`。金额使用 `BigDecimal`，CSV 读写使用 Apache Commons CSV，项目使用 Gradle 和 JUnit 5。

## User Stories

1. 作为用户，我想通过命令新增一笔支出，以便记录一次日常消费。
2. 作为用户，我想通过命令新增一笔收入，以便记录工资、奖金或其他入账。
3. 作为用户，我想为每笔交易指定分类，以便后续知道钱花在哪里或从哪里来。
4. 作为用户，我想为每笔交易指定发生日期，以便周汇总和月汇总按真实发生时间统计。
5. 作为用户，我想为交易添加可选备注，以便记录额外上下文。
6. 作为用户，我想在备注中输入逗号、引号或换行，以便自然地记录说明而不破坏 CSV 数据。
7. 作为用户，我想金额只能接受大于 0 的值，以便避免无意义或错误交易进入系统。
8. 作为用户，我想金额最多保留 2 位小数，以便交易金额符合日常记账习惯。
9. 作为用户，我想程序统一以 2 位小数展示金额，以便列表和汇总结果清晰一致。
10. 作为用户，我想新增交易后数据保存在本地，以便程序退出后再次运行仍能读取记录。
11. 作为用户，我想查看最近交易列表，以便确认刚录入的数据是否正确。
12. 作为用户，我想每笔交易都有唯一 `id`，以便准确删除指定交易。
13. 作为用户，我想删除一笔录错的交易，以便修正错误数据并保证汇总可信。
14. 作为用户，我想删除不存在的 `id` 时收到明确错误提示，以便知道没有交易被删除。
15. 作为用户，我想删除成功后看到被删除交易的简短信息，以便确认删除的是正确记录。
16. 作为用户，我想查看某个日期所在 ISO 周的汇总，以便了解这一周的收入、支出和净收入。
17. 作为用户，我想周汇总按周一到周日统计，以便每次运行都得到稳定结果。
18. 作为用户，我想查看某个自然月的汇总，以便了解整月的收入、支出和净收入。
19. 作为用户，我想汇总中看到总收入、总支出和净收入，以便快速判断收支状况。
20. 作为用户，我想汇总中看到按分类分组的支出小计，以便知道主要消费类别。
21. 作为用户，我想在参数缺失时看到清晰用法说明，以便快速修正命令。
22. 作为用户，我想在交易类型非法时收到错误提示，以便只录入 `income` 或 `expense`。
23. 作为用户，我想在日期格式非法时收到错误提示，以便修正日期输入。
24. 作为用户，我想在分类为空时收到错误提示，以便每笔交易都能被归类。
25. 作为用户，我想运行 `help` 或无参数命令时看到可用命令列表，以便知道如何使用工具。
26. 作为开发者，我想领域逻辑不依赖 CLI 输入输出，以便可以单独测试校验和汇总规则。
27. 作为开发者，我想 CSV 读写被封装在 infrastructure 层，以便 Apache Commons CSV 不泄漏到领域模型。
28. 作为开发者，我想通过 JUnit 5 覆盖金额、日期、CSV、汇总和用例行为，以便实现可以安全演进。
29. 作为学习者，我想项目保持简单分层而不是引入应用框架，以便练习 Java 基础和设计边界。
30. 作为维护者，我想 v1 明确不支持编辑、账户、币种、标签和多用户，以便控制第一版范围。

## Implementation Decisions

- v1 使用 Java 实现。
- v1 使用 Gradle 作为构建工具，JUnit 5 作为测试框架。
- v1 不引入 Spring、Micronaut、Quarkus 等应用框架。
- v1 使用简单分层包结构：`cli`、`domain`、`application`、`infrastructure`。
- `cli` 负责参数解析、输出文本和退出码。
- `domain` 负责 `Transaction`、`TransactionType`、金额和日期校验、汇总模型。
- `application` 负责新增、删除、列表、汇总等用例服务。
- `infrastructure` 负责 CSV 文件读写、ID 生成和系统时间适配。
- v1 使用非交互式参数命令，不进入交互式问答。
- v1 命令包括：新增交易、查看列表、删除交易、查看周汇总、查看月汇总、查看帮助。
- 参数缺失或非法时，命令应失败并打印清晰用法说明。
- 交易字段固定为 `id`、`type`、`amount`、`category`、`date`、`note`、`createdAt`。
- 交易类型只支持 `income` 和 `expense`。
- v1 不加入账户、币种、标签、多用户字段。
- 金额使用 `BigDecimal` 表示，不使用 `double` 或 `float` 参与金额计算。
- 输入金额必须大于 `0`，最多保留 2 位小数。
- 输出金额统一显示 2 位小数。
- v1 不处理币种，金额是无币种数值。
- v1 使用 CSV 作为主存储格式，不只是导入导出格式。
- CSV 存储文件是新增、删除、列表、周汇总和月汇总的数据源。
- CSV 读写使用 Apache Commons CSV。
- CSV 读写必须正确处理逗号、引号和换行转义，不能使用简单字符串拆分实现。
- Apache Commons CSV 依赖只能存在于 CSV 读写适配层，不能泄漏到领域模型和汇总逻辑。
- 周汇总使用 ISO 周：周一是一周开始，周日是一周结束。
- 月汇总使用自然月。
- 周汇总和月汇总至少包含总收入、总支出、净收入，以及按分类分组的支出小计。
- v1 支持删除交易但不支持编辑交易。
- 删除交易使用物理删除：从 CSV 存储文件中移除记录，不使用软删除字段。
- 删除不存在的 `id` 时命令失败，并提示未找到对应交易。
- 删除成功后输出被删除交易的简短信息。
- 列表默认显示最近交易，过滤参数留到后续版本。
- `help` 或无参数时展示可用命令。

## Testing Decisions

- 测试应优先覆盖外部行为，不测试实现细节。
- 最高优先级测试 seam 是 CLI 行为 seam：通过运行命令验证新增、持久化、列表、删除、周汇总、月汇总、帮助信息和错误提示。
- application 用例 seam 用于测试新增、删除、列表、汇总等业务流程，可以使用临时 CSV 或 fake repository。
- domain seam 用于测试纯逻辑：金额校验、交易类型校验、日期范围计算、ISO 周范围、月范围、汇总计算。
- infrastructure CSV seam 用于测试 CSV 读写边界，尤其是备注包含逗号、引号和换行的场景。
- 金额测试必须覆盖大于 0、0、负数、超过 2 位小数、输出 2 位小数等情况。
- 日期测试必须覆盖合法日期、非法日期、ISO 周周一到周日范围、跨月周范围、自然月范围。
- 删除测试必须覆盖删除存在交易、删除不存在 `id`、删除后 CSV 重写、删除后汇总不再包含该交易。
- 汇总测试必须覆盖收入、支出、净收入、按分类分组的支出小计。
- CLI 错误测试必须覆盖参数缺失、非法交易类型、非法金额、非法日期、空分类。
- CSV 测试必须确认读写结果能 round-trip，不损坏备注中的特殊字符。
- 测试不应断言内部私有方法或具体实现细节；应通过领域对象、用例返回值、CSV 内容或 CLI 输出验证行为。
- 当前仓库没有已有源码或测试先例，因此 v1 需要建立第一套测试约定。

## Out of Scope

- v1 不支持编辑已有交易。
- v1 不支持账户管理。
- v1 不支持币种、汇率或本地化金额格式。
- v1 不支持标签或多维分类。
- v1 不支持多用户。
- v1 不支持数据库、远程同步或云存储。
- v1 不支持软删除、恢复删除或删除审计。
- v1 不支持预算限制。
- v1 不支持周期性支出。
- v1 不支持搜索和过滤参数，列表仅默认显示最近交易。
- v1 不支持命令历史。
- v1 不提供交互式问答输入模式。
- v1 不引入应用框架或复杂架构模式。
- v1 不要求完整 CSV 导入导出命令；CSV 主存储和正确读写是第一阶段重点。

## Further Notes

本 PRD 尊重当前已接受的 ADR：v1 支持删除但不支持编辑、使用 CSV 作为本地存储、固定交易字段、使用非交互式参数命令、周汇总使用 ISO 周、金额使用 BigDecimal、使用 Gradle 和 JUnit 5、使用 Apache Commons CSV、使用简单分层包结构、删除交易使用物理删除。

后续可以从本 PRD 拆分 vertical slice issues。优先切片应从可运行项目骨架和最小新增/list 闭环开始，再逐步加入删除、汇总、CSV 边界和错误处理。
